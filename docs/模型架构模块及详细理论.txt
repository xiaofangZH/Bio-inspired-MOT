为了应对通用多目标跟踪中搜索效率低下和内存易受污染两大核心挑战，我们提出了分层记忆增强追踪器（ Himatr），一个端到端的统一Transformer架构。如图2所示， Himatr遵循一个循环处理流程，在每个时间步t接收当前帧It和过去时刻的记忆状态Mt−k，输出当前帧的追踪结果Ot和更新后的记忆Mt。整个架构由三个紧密耦合的阶段组成：1）分层视觉编码器；2）记忆增强解码器；3）循环记忆更新。
给定视频序列的第一帧I1和一个初始目标框b1，我们首先初始化一个代表该目标的记忆状态m1。从第二帧开始，对于每个时间步t， Himatr执行编码-解码-更新三个阶段的操作。在编码阶段，分层视觉编码器将当前帧It编码为一组包含全局上下文和局部细节的图像特征Ft。在进行解码时，记忆增强解码器接收来自历史记忆Mt−k的追踪查询（Track Queries）和一组可学习的检测查询（Detect Queries），并让它们与图像特征Ft和历史记忆Mt−k进行交互，最终预测出每个目标的位置、类别和掩膜。在最后的更新阶段，循环记忆更新模块将当前帧的预测结果与旧的记忆状态Mt−k进行融合，生成用于下一帧的新记忆状态Mt。
3.1 分层视觉编码器 (Hierarchical Visual Encoder)
传统的Transformer跟踪器对整个搜索区域进行统一的、高成本的注意力计算，这对于在广阔、杂乱的日常背景中搜索小目标是低效的。Swin Transformer [39]通过引入移动窗口和层级化设计，将注意力计算限制在局部窗口内，显著降低了计算量。我们的分层视觉编码器也借鉴了这种层级化的思想，同时采用生物视觉中眼跳扫视的策略，以一种“由粗到精”的方式高效地分配计算资源。
该编码器包含一个共享的ViT骨干网络（由DINOv3预训练权重初始化）和两条并行的处理路径：
3.1.1 全局-局部扫描路径 (Peripheral Scan-Foveal Focus Path)
我们将当前帧It缩放到一个较低的分辨率It',它被切分成patch并送入ViT骨干的前L层，得到一组低分辨率的特征Tokens Flow。一个轻量级的区域提议网络（RPN-lite），由几个卷积层构成，作用于Flow重塑后的特征图上，预测出K个最有可能包含目标的候选区域（Regions of Interest, RoIs）。
It∈ℝH×W×3
It'∈ℝH'×W'×3
Flow∈ℝNlow×d
R=r1,r2,...,rK
Fhigh∈ℝNhigh×d
我们根据RPN-lite预测的RoIs R，从原始高分辨率图像It中裁切出对应的K个图像块。这些高分辨率图像块被独立地切分成patch，并线性投影成一组高分辨率的特征Tokens Fhigh。
3.1.2 特征融合与输出
我们将低分辨率特征Tokens Flow和高分辨率特征Tokens Fhigh进行拼接，形成一个融合了全局和局部信息的特征表示，并送入ViT骨干的后M−L层进行深度融合。
Ft=ViTdeeper(Concat(Flow,Fhigh))
Ft∈ℝ(Nlow+Nhigh)×d
其中Ft是最终输出的、融合了全局上下文和局部细节的图像特征。这种设计使得后续的解码器能够在一个浓缩的、信息丰富的特征空间中进行操作，而不是在冗余的原始像素空间中,为后续的跟踪任务提供了更为丰富和准确的特征基础。通过这种方式，我们的模型能够在复杂场景中更有效地捕捉目标的位置和状态变化。
3.2 记忆增强解码器 (Memory-Augmented Decoder)
这是 Himatr的核心创新所在，目的是解决“内存污染”问题。其核心是一个新颖的门控记忆注意力（Gated Memory Attention）机制，它使模型能够自适应地、智能地融合来自当前观测和历史记忆的信息。解码器遵循标准的Transformer设计，由多层堆叠而成。在每一层中，除了标准的自注意力模块（用于查询间的交互）和前馈网络（FFN）外，其交叉注意力模块被我们提出的门控记忆注意力所取代。
3.2.1 门控记忆注意力 (Gated Memory Attention)
3.2.1.1 并行信息提取 (Parallel Information Extraction)
在解码器的一层中，我们假设有Qdec（查询），代表Nq个待追踪的目标（包含Track Queries和Detect Queries）；当前帧的图像特征Ft（来自公式）；过去k帧的历史记忆Mt−k（来自公式）。我们首先使用图像交叉注意力ICA（Image Cross-Attention）与记忆交叉注意力MCA（Memory Cross-Attention）让查询Qdec并行地与当前帧的视觉证据和历史记忆进行交互，分别计算出两种“意见”。图像交叉注意力负责提取目标在当前帧中的视觉线索，记忆交叉注意力负责从历史中“回忆”目标的稳定特征。

Qdec∈ℝNq×d
Ft∈ℝNp×d
Mt−1∈ℝNm×d
Oimg=softmaxQdec(FtWKimg)Td(FtWVimg)
Omem=softmaxQdec(Mt−1WKmem)Td(Mt−1WVmem)
其中，WKimg，WVimg，WKmem，WVmem是可学习的投影矩阵。Oimg代表了“眼见为实”的信息，Omem代表了“历史经验”的判断。通过这种方式，模型能够同时考虑当前帧的视觉证据和历史记忆中的信息，从而更准确地跟踪目标。具体来说，Oimg和Omem的加权和将作为解码器的输入，用于生成最终的跟踪结果。这种设计使得模型在处理复杂场景或目标遮挡时，能够利用历史记忆中的稳定特征来增强跟踪的鲁棒性。
3.2.1.2自适应门控融合 (Adaptive Gated Fusion)
在研究过程中我们发现，简单地将Oimg和Omem相加或拼接，无法解决信息冲突问题。例如，当Oimg因特征退化而“有噪声”时，它会污染更可靠的Omem。因此，我们设计一个动态门控（Dynamic Gate）g，该门控由查询Qdec自身的状态以及两种信息源的差异共同决定，用于智能地平衡两者。
g=σ(MLP([Qdec,Oimg−Omem,Oimg⊙Omem]))
g∈(0,1)Nq×d
其中，[⋅,⋅]表示拼接操作，⊙表示哈达玛积，MLP是一个小型的多层感知机，σ是Sigmoid函数，确保门控值g处于合理范围内。通过这种多维度门控的设计，Qdec提供了目标自身的状态；Oimg−Omem显式地衡量了“当前观测”与“历史记忆”之间的冲突程度（divergence），当冲突大时（例如，目标被遮挡，当前观测是背景），门控需要做出更谨慎的判断；Oimg⊙Omem则衡量了两者特征的一致性（agreement）。通过MLP学习这三者与最终权重之间的非线性关系，使得门控决策极为灵活和强大。
在对门控进行计算后，我们对信息进行融合。该融合操作允许模型在每个特征维度上进行加权。当目标清晰可见、Oimg可靠时，模型可以学习给g赋予一个接近1的值；当目标被遮挡或特征退化、Oimg不可靠时，模型则学习给g赋予一个接近0的值，从而更多地依赖从历史记忆Omem中提取的稳定特征，有效阻止了“坏”特征对更新过程的污染。
解码器的最终输出经过前馈网络（FFN）处理后，被送入并行的预测头，回归出目标的边界框（Bounding Box）、类别（Classification）和分割掩膜（Mask）。
3.2.2 循环记忆更新 (Recurrent Memory Update)
为了维持记忆的长期稳定性和时序连贯性，我们采用基于门控循环单元（GRU）的机制来更新记忆库。对于一个在当前帧被成功匹配的物体i，我们有它在记忆库中的旧特征向量mt−1i和它在当前帧解码器中输出的新特征向量oti。新的记忆向量Mti通过以下GRU更新法则计算：
Mt−1i∈ℝd
oti∈ℝd
zt=σ(Wz[oti,Mt−1i])
rt=σ(Wr[oti,Mt−1i])
Mti=tanh(Wh[oti,rt⊙Mt−1i])
Mti=(1−zt)⊙Mt−1i+zt⊙Mti
其中，Wz、Wr、Wh是GRU可学习的参数，这种循环更新机制能够平滑地演化目标的特征表示，有效滤除单帧的观测噪声，保留了对目标身份最本质的长期记忆。对于未被匹配的旧记忆，我们会根据其“年龄”进行逐步衰减和最终的清理；对于新检测到的目标，则在记忆库中为其创建新的条目。
这种处理方式确保了记忆库的动态更新，既不会因长期未更新的旧记忆而占用过多资源，又能及时纳入新出现的目标信息。同时，通过设定合理的衰减策略，可以平衡记忆的时效性和稳定性，避免因记忆过快衰减而导致的跟踪丢失问题。此外，记忆库的容量管理也是关键，需要设计合理的机制来防止其无限增长，同时保证重要信息的保留。
